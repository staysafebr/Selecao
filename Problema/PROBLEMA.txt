Introdução

A Internet das Coisas ou Internet of Things (IoF), é um termo  utilizado 
para designar a conectividade entre vários tipos de objetos do dia a dia   
sensíveis à internet, desde de eletrodomésticos até outros  equipamentos  
espalhados pela cidade. [2] 

*** Exemplo: Um Beaglebone Black conectado a  um leitor de RFID. 

Conceitualmente,  é  a  posibilidade  de conectar o  mundo  físico com o 
mundo  digital por  meio da  web.   Assim, com  o mundo  cada  vez  mais  
interconectado,  não  apenas  as pessoas se  comunicam,  mas  "cada  vez 
mais os objetos estão "conversando" uns com os outros". [2] 

*** Exemplo: Protocolo MQTT para comunicação.

A Internet das Coisas é  a  conexão de todas as  máquinas, dispositivos, 
sensores, automóveis, câmeras e  outras "coisas" que  buscam  otimizar a 
vida das pessoas  e  organizações, "além de melhorar operações do dia  a  
dia  e  a  economizar tempo,  dinheiro  e  até  mesmo vidas". [2]

E essa ampliação da conectividade é particularmente útil para uma  série 
de atividades rotineiras. Os  objetos que  estiverem em rede conseguirão 
fornecer dados para  sistemas  integrados  e  controlar   uma   série de 
atividades. [2]

*** Exemplo: Realizar leituras programadas através de um leitor RFID.
	     
Há  várias   ferramentas   capazes de  armazenar a história dos objetos, 
como os velhos  códigos de  barra ou os  sensores  wireless,  mas a  que 
vem  sendo  mais  utilizada na  pesquisa de  Internet  das  Coisas  é  a 
tecnologia   RFID   (Identificação por Radiofrequência),  que   rastreia 
coisas  por  meio  de   ondas  de  rádio e,  geralmente,  é acoplada aos 
objetos por meio de uma simples etiqueta. [1]

Porém   muito  ainda   precisa  ser feito para consolidar a  comunicação 
inteligente de  máquina para   máquina.  O  maior  desafio  é  a conexão 
entre os objetos e uma rede externa onde eles possam buscar dados. [1]

O Problema

Você é um administrador de  sistemas e  possui diversos  leitores de RFID 
espalhados  pelo  Brasil. Estes leitores  estão em empresas  diferentes e 
possuem configurações diferentes. Por exemplo: Você pode ter um leitor em
Tucuruí/PA e um em Goiânia/GO, e  estes leitores  possuem IP diferentes e
configurações   diferentes, o  de  Tucuruí utiliza  um   modo de  leitura 
"Transparent" enquanto o de Goiânia utiliza o modo "HID".

Estes   leitores   estão  ligados  a um    sistema  embarcado. O  sistema 
embarcado utilizado é o Beaglebone Black. Você  possui acesso SSH a todos
os  sistemas  embarcados e  assim  pode  configurar  os leitores  de RFID.

A empresa  onde você  trabalha assinou um contrato de prestação de serviço
com uma  grande  empresa  produtora  de vinhos e  agora você possui  3.500 
leitores para monitorar e configurar diariamente. 

O acesso SSH se tornou  inviavel  para configuração  dos  leitores  e você 
precisa   criar  uma   interface WEB  utilizando o  framework  Django para 
cadastrar e configurar os leitores de RFID.

Para realizar esta tarefa você fez algumas  anotações, e  definiu  que irá 
utilizar no  mínimo três  classes, que  são leitor,  configuração leitor e 
localidade.

Anotações:

	[1] - Crie uma interface para cadastro de leitores de RFID.
	      	Os dados utilizados são: 
			[1.1] IP (Exemplo: 192.168.1.100).
			[1.2] Porta (Exemplo: 1842).
			[1.3] Região (Exemplo: Goiânia/GO).
			[1.4] Identificador p/ Arquivo de Configuração (Foreign Key).  
	
	[3] - Crie uma interface para cadastro de configurações.
	      	Os dados utilizados são: 
	      		[3.1] Tipo de modo de leitura (Transparent ou HID).
 			[3.2] Região do leitor (Brazil [13] ou USA [14]).
			[3.3] Algoritmo EPC (Single Cycle [SS] ou Raw [RW]).
			[3.4] ID do Leitor Fisico.
